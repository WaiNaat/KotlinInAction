# 연산자 오버로딩과 다른 관례 (Operator Overloading and Other Conventions)

Kotlin은 단순히 문법 설탕(Syntactic Sugar)을 제공하는 언어가 아니다.
특정 이름의 함수에 의미를 부여하는 **Convention 기반 언어 설계**를 사용한다.

즉:

```kotlin
a + b
```

는 실제로:

```kotlin
a.plus(b)
```

로 변환된다.

* Kotlin의 연산자는 결국 함수 호출이다
* Kotlin은 convention 기반으로 다양한 문법을 확장한다
* 연산자 오버로딩은 DSL, Collection, 수학 객체 등에 매우 강력하다
* 하지만 과도한 사용은 가독성을 망친다

---

# 1. 연산자 오버로딩(Operator Overloading)

Kotlin은 특정 함수 이름에 `operator` 키워드를 붙이면 연산자로 사용할 수 있다.

예시:

```kotlin
data class Point(val x: Int, val y: Int)

operator fun Point.plus(other: Point): Point {
    return Point(x + other.x, y + other.y)
}
```

사용:

```kotlin
val p1 = Point(10, 20)
val p2 = Point(30, 40)

println(p1 + p2)
```

실제 컴파일:

```kotlin
p1.plus(p2)
```

---

# 2. 산술 연산자 오버로딩

지원되는 주요 연산자:

| 연산자       | 함수명            |
| --------- | -------------- |
| `+`       | `plus()`       |
| `-`       | `minus()`      |
| `*`       | `times()`      |
| `/`       | `div()`        |
| `%`       | `mod()`        |
| unary `-` | `unaryMinus()` |

---

## 예시

```kotlin
operator fun Point.times(scale: Int): Point {
    return Point(x * scale, y * scale)
}

println(Point(10, 20) * 2)
```

# 3. 복합 대입 연산자 (`+=`, `-=`)

Kotlin은 두 가지 방식으로 처리한다.

## 1) `plusAssign()`

```kotlin
operator fun MutableList<Int>.plusAssign(value: Int) {
    add(value)
}
```

사용:

```kotlin
list += 3
```

---

## 2) `plus()`

만약 `plusAssign()`이 없으면:

```kotlin
a += b
```

는:

```kotlin
a = a + b
```

로 변환된다.

---

# 4. 비교 연산자

## `==`

Kotlin의 `==` 는 구조적 동등성(structural equality)

```kotlin
a == b
```

↓

```kotlin
a?.equals(b) ?: (b == null)
```

즉 Java의:

```java
Objects.equals(a, b)
```

와 비슷하다.

---

## `===`

참조 동일성(reference equality)

```kotlin
a === b
```

메모리 주소 비교.

# 5. Comparable과 순서 비교

```kotlin
operator fun compareTo(other: Person): Int
```

를 구현하면:

```kotlin
p1 > p2
```

가능.

예시:

```kotlin
class Person(val age: Int) : Comparable<Person> {
    override operator fun compareTo(other: Person): Int {
        return age.compareTo(other.age)
    }
}
```

# 6. 컬렉션 접근 연산자 (`[]`)

## get

```kotlin
operator fun get(index: Int)
```

↓

```kotlin
obj[index]
```

---

## set

```kotlin
operator fun set(index: Int, value: T)
```

↓

```kotlin
obj[index] = value
```

---

## 예시

```kotlin
data class Point(val x: Int, val y: Int)

operator fun Point.get(index: Int): Int {
    return when(index) {
        0 -> x
        1 -> y
        else -> throw IndexOutOfBoundsException()
    }
}
```

사용:

```kotlin
println(point[0])
```

# 7. `in` 연산자

```kotlin
x in collection
```

↓

```kotlin
collection.contains(x)
```

예시:

```kotlin
operator fun Rectangle.contains(p: Point): Boolean {
    return p.x in upperLeft.x until lowerRight.x &&
           p.y in upperLeft.y until lowerRight.y
}
```

# 8. Range와 `..`

```kotlin
1..10
```

↓

```kotlin
1.rangeTo(10)
```

---

## 중요한 점

`..` 는 `ClosedRange`

즉 끝값 포함.

```kotlin
1..5
```

→ 1,2,3,4,5

---

## until

끝값 제외:

```kotlin
0 until 10
```

→ 0~9

실무에서 훨씬 많이 사용.

# 9. destructuring declaration

```kotlin
val (x, y) = point
```

↓

```kotlin
val x = point.component1()
val y = point.component2()
```

---

## data class

자동 생성됨.

```kotlin
data class Point(val x: Int, val y: Int)
```

---

## 직접 구현 가능

```kotlin
operator fun Point.component1() = x
operator fun Point.component2() = y
```

# 10. 반복문 관례 (`iterator`)

```kotlin
for (x in collection)
```

↓

```kotlin
collection.iterator()
```

그리고:

```kotlin
iterator.next()
iterator.hasNext()
```

사용.

# 11. invoke 관례

객체를 함수처럼 호출 가능.

```kotlin
class Greeter {
    operator fun invoke(name: String) {
        println("Hello $name")
    }
}
```

사용:

```kotlin
val greeter = Greeter()

greeter("Kotlin")
```

↓

```kotlin
greeter.invoke("Kotlin")
```

---

# 12. 위임 프로퍼티(delegated property)

Kotlin 관례의 핵심 중 하나.

```kotlin
val p by delegate
```

↓

```kotlin
delegate.getValue(...)
delegate.setValue(...)
```

---

## lazy

대표 사례:

```kotlin
val value by lazy {
    expensiveComputation()
}
```

# 13. Convention 기반 언어 설계

Kotlin은 새로운 문법을 추가하지 않고 convention으로 기능을 확장한다.

예:

| 문법            | 실제 호출           |
| ------------- | --------------- |
| `a + b`       | `a.plus(b)`     |
| `a[i]`        | `a.get(i)`      |
| `a()`         | `a.invoke()`    |
| `for(x in y)` | `y.iterator()`  |
| `x in y`      | `y.contains(x)` |
