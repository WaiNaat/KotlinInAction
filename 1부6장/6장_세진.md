# 6장. 컬렉션과 시퀀스

Kotlin Collection API는 단순 유틸 함수가 아니라 **데이터를 선언적으로 처리하는 함수형 API**이며 대부분 내부적으로 iterator loop + inline 함수 구조로 구현되어 있다. 성능은 대부분 O(n)이며 실제 차이는 **몇 번 순회하는지, 중간 Collection 생성 여부, short-circuit 가능 여부**에서 발생한다.

Collection은 eager evaluation이라 chain마다 Collection이 생성되고, Sequence는 lazy evaluation이라 element 단위 pipeline 처리된다.

예:

```kotlin
val result = listOf(1,2,3,4,5)
    .filter { it > 2 }
    .map { it * 10 }
```

내부적으로:

```
filter → 새 List 생성 → map → 또 List 생성
```

Sequence:

```kotlin
val result = listOf(1,2,3,4,5)
    .asSequence()
    .filter { it > 2 }
    .map { it * 10 }
    .toList()
```

동작:

```
1 → filter → map
2 → filter → map
3 → filter → map
```

즉 순회 1번.

---

# 6.1 컬렉션에 대한 함수형 API

대부분 함수:

| 함수        | 시간복잡도 | 특징            |
| --------- | ----- | ------------- |
| filter    | O(n)  | 새 List 생성     |
| map       | O(n)  | 변환            |
| count     | O(n)  | 메모리 추가 없음     |
| find      | O(n)  | short circuit |
| groupBy   | O(n)  | Map + List 생성 |
| associate | O(n)  | Map만 생성       |

---

# 6.1.1 filter와 map

filter는 조건 만족 element만 유지.

```kotlin
val numbers = listOf(1,2,3,4,5)

val result = numbers.filter {
    it % 2 == 0
}

println(result)
// [2,4]
```

내부 동작:

```kotlin
val result = ArrayList<Int>()

for(n in numbers){
    if(n % 2 == 0){
        result.add(n)
    }
}
```

시간복잡도:

```
O(n)
```

map:

```kotlin
val result = numbers.map {
    it * 2
}

println(result)
// [2,4,6,8,10]
```

중요한 실무 포인트는 연산 순서다.

비효율:

```kotlin
numbers
    .map { it * 100 }
    .filter { it > 200 }
```

효율적:

```kotlin
numbers
    .filter { it > 2 }
    .map { it * 100 }
```

이유는 filter로 연산 대상 줄이면 map 호출 횟수 감소하기 때문.

# 6.1.2 reduce와 fold

reduce는 첫 element를 accumulator로 사용한다.

```kotlin
val numbers = listOf(1,2,3,4)

val sum = numbers.reduce { acc, element ->
    acc + element
}

println(sum)
// 10
```

파라미터 의미:

| 파라미터    | 의미         |
| ------- | ---------- |
| acc     | 현재 누적 값    |
| element | 현재 element |

동작:

```
acc = 1
1+2=3
3+3=6
6+4=10
```

문제:

empty면 exception 발생.

```kotlin
emptyList<Int>().reduce { acc,e ->
    acc+e
}
```

그래서 fold가 더 안전하다.

```kotlin
val sum = numbers.fold(100){
    acc, element -> acc + element
}

println(sum)
// 110
```

fold 특징:

| 특징         | 설명         |
| ---------- | ---------- |
| initial 존재 | empty safe |
| 타입 변경 가능   | 매우 중요      |

예:

```kotlin
val text = numbers.fold(""){
    acc,e -> acc + e
}
```

실무에서는 fold가 더 많이 사용된다.

# 6.1.5 groupBy

key 기준 grouping.

```kotlin
data class Person(
    val name:String,
    val age:Int
)

val people=listOf(
    Person("A",20),
    Person("B",20),
    Person("C",30)
)

val result=people.groupBy {
    it.age
}
```

결과:

```
Map<Int,List<Person>>
```

주의:

메모리 많이 사용.

key당 1개면 associateBy가 더 좋다.

# 6.1.6 associate

associateBy:

```kotlin
val map=people.associateBy {
    it.name
}
```

associateWith:

```kotlin
val map=numbers.associateWith {
    it*10
}
```

associate:

```kotlin
val map=numbers.associate {
    it to it*10
}
```

groupBy와 차이 (메모리 더 효율적)
- groupBy   : `Map<K,List<V>> `
- associate : `Map<K,V>`

# 6.2 시퀀스

Sequence는 lazy evaluation이다.

예:

```kotlin
val result=numbers
    .asSequence()
    .filter{
        println("filter $it")
        it>2
    }
    .map{
        println("map $it")
        it*10
    }
    .first()
```

출력:

```
filter 1
filter 2
filter 3
map 3
```

3 찾으면 종료.

Collection이면:

```
filter 전체 → map 전체
```

Sequence 사용 이유:

| 이유                | 설명    |
| ----------------- | ----- |
| 중간 객체 감소          | GC 감소 |
| early termination | 빠른 종료 |
| 대용량 데이터           | 효율적   |

주의:

작은 데이터는 오히려 느릴 수 있다 (iterator overhead 때문).

# 6.2.1 중간 연산과 최종 연산

중간:

```kotlin
sequence.filter{}

sequence.map{}
```

Sequence 반환.

최종:

```kotlin
sequence.count()

sequence.first()

sequence.any()

sequence.find()

sequence.toList()
```

판단 기준:

```
Sequence 반환 → 중간
다른 타입 반환 → 최종
```

toList만 terminal 아니다.

# 시퀀스 사용시 주의

lazy라 실행 시점 값 사용.

```kotlin
val list= mutableListOf(1,2,3)

val seq=list.asSequence()
    .map{it*2}

list.add(4)

println(seq.toList())

// [2,4,6,8]
```

또 single iteration 개념이라 재사용 주의.
