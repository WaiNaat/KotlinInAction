## 6.1 컬렉션에 대한 함수형 API

## 6.1.1 원소 제거와 변환 filter와 map

컬렉션을 다룰 때 원소를 제거하기 위해 filter 변환을 위해 map 함수를 사용할 수 있다.

```kotlin
fun main() {
	val list = listOf(1,2,3,4)
	println(list.filter( it % 2 == 0 }) // 2, 4
	
	val people = listOf(Person("Alice",29), Person("Bob",31))
	println(people.map( it.name }) // Alice, Bob
}
```

함수안에 넣는 람다를 술어라고 한다.

maxByOrNull같은 함수도 사용할 수 있다.

```kotlin
people.filter {
	val oldestPerseon = people.maxByOrNull(Person::age)
	it.age == oldestPerson?.age
}

val maxAge = people.maxByOrNull(Person::age)?.age
people.filter {it.age == maxAge}
```

만약 인덱스도 필요하다면 filterIndexed, mapIndexed도 사용가능하다

```kotlin
fun main() {
	val numbers = listOf(1, 2, 3, 4, 5, 6, 7)
	val filtered = numbers.filterIndexed { index, element ->
		index % 2 == 0 && element > 3
	}
	// 5, 7
	
	val mapped = numbers.mapIndexed { index, element ->
		index + element
	}
	// 1, 3,5 7, 9, 11, 13
}
```

맵의 경우 filterKeys, mapKeys등을 사용할 수 있다.

## 6.1.2 컬렉션 값 누적: reduce와 fold

reduce와 fold는 컬렉션으로 한 값을 받아서 누적할 때 사용한다

reduce를 사용해 첫 값을 누적기에 넣고 지속하여 값을 전달한다고 해보자.

```kotlin
fun main() {
	val list = listOf(1, 2, 3, 4)
	println(list.reduce { acc, element ->
		acc + element
		})
	// 10
	println(list.reduce { acc, element ->
		acc * element
		})
	// 24
}
```

첫번째 원소가 누적되어 계산된 후 최종값이 반환된다.

fold를 사용하면 첫번쨰 원소 대신 임의의 값을 첫번째 값으로 사용할 수 있다.

```kotlin
fun main() {
	val people = listOf(
		Person("Alex", 29),
		Person("Bob", 28)		
	)
	val folded = people.fold("") { acc, person ->
		acc + person.name
		}
	println(folded)
	// AlexBob
}
```

만약 누적되는 중간단계의 모든 값을 뽑아내고 싶으면 runningReduce, runningFold를 사용하면 된다.

```kotlin
fun main() {
	val list = listOf(1, 2, 3 ,4)
	val summed = list.runningReduce { acc, element ->
		acc + element
	}
	println(summed)
	// 1 3 6 10
	val multipled = list.runningFold(2) { acc, element ->
		acc * element
	}
	println(multipled)
	// 2 6 36 360
}
```

## 6.1.3 커렉션에 술어 적용: all, any, none, count, find

all : 모든 원소가 이 술어를 만족하는지 궁금하다면 사용한다. 만약 empty면 무조건 true이다.

```kotlin
fun main() {
	val people = listOf(Person("Alice", 27), Person("Bob", 31))
	println(people.all(canBeInClub27))
	// false
}
```

any : 만족하는 원소가 하나라도 있으면 true이다. 만약 empty이면 무조건 false이다.

```kotlin
fun main() {
	val people = listOf(Person("Alice", 27), Person("Bob", 31))
	println(people.all(canBeInClub27))
	// true
}
```

none : 만족하는 원소가 하나도 없다. 즉 !any와 마찬가지. empty이면 무조건 true이다

count : 만족하는 원소의 개수를 알고 싶을 떄 사용한다

```kotlin
fun main() {
	val people = listOf(Person("Alice", 27), Person("Bob", 31))
	println(people.count(canBeInClub27))
	// 1
}
```

<aside>
💡

**함수를 적재적소에 사용하라: count 와 size**

people.filter(canBeInClub27).size

people.count(canBeInClub27)

두개의 차이는 전자는 컬렉션 객체가 하나생기고 그 컬렉션 객체의 size를 반환한다. 반면 count는 숫자만 갖고 있고 따로 원소에 대한 정보는 갖고 있지 않는다.

</aside>

find: 술어를 만족하는 원소를 하나 찾고 싶으면 사용한다. 2개이상이면 첫째 사용한다(firstOrNull)과 동일

```kotlin
fun main() {
	val people = listOf(Person("Alice", 27), Person("Bob", 31))
	println(people.find(canBeInClub27))
	// Alice 27
	
	val people2 = listOf(Person("Bob", 31))
	println(people2.find(canBeInClub27))
	// null
}
```

## 6.1.4 리스트를 분할해 리스트의 쌍으로 만들기: partition

컬렉션에서 어떤 술어를 만족하는 그룹과 아닌 그룹으로 나누기 위해 partition을 사용할 수 있다.

```kotlin
val (comeIn, stayOut) = people.partition(canBeInClub27)
println(comeIn) // Alice 27
println(stayOut) // Bob 31
```

## 6.1.5 리스트를 여러 그룹으로 이뤄진 맵으로 바꾸기: groupBy

때에 따라 참, 거짓으로만 분리할 수 없고 특성에 따라 나눠야 할 때도 있다. 이때 groupBy를 사용한다.

```kotlin
fun main() {
	val people = listOf(
	 Person("Alice", 27), 
	 Person("Bob", 31),
	 Person("Carol", 31)	 
	)
	
	println(people.groupBy {it.age} )
	// 결과 타입 Map<Int, List<Person>>
	// 31 = [Bob 31, Carol 31] , 27 = [Alice 27]
}
```

## 6.1.6 컬렉션을 맵으로 변환: associate, associateWith, associateBy

원소를 그룹화하지 않으면서 컬렉션으로부터 맵을 만들어내고 싶다면 associate함수를 사용한다.

만약 같은 키를 사용하게 된다면 마지막 값으로 저장된다(덮어쓰기 방식)

```kotlin
fun main() {
	val people = listOf(
	 Person("Alice", 27), 
	 Person("Bob", 31),
	 Person("Carol", 31)	 
	)
	val personToAge = people.associateWith { it.age }
	println(personToAge)
	// {Alice 27 = 27, Bob 31 = 31, Carol 31 = 31
	
	val ageToPerson = people.associateBy { it.age }
	// {27 = Alice 27, 31 = Carol 31
}
```

## 6.1.7 가변 컬렉션의 원소 변경: replaceAll, fill

replaceAll : 원소를 람다의 값으로 모두 바꿈

fill : 원소를 모두 같은 하나의 값으로 바꿈

```kotlin
fun main() {
	val names = mutableListOf("Martin", "Samuel")
	
	names.replaceAll { it.uppercase() }
	println(names) // MARTIN, SAMUEL
	
	names.fill("redacted")
	println(names) // redacted, redacted
}
```

## 6.1.8 컬렉션의 특별한 경우 처리: ifEmpty

ifEmpty를 사용하면 컬렉션에 아무 원소도 없을 때 기본값을 생성하는 람다를 제공할 수 있다.

```kotlin
fun main() {
	val empty = emptyList<String>()
	val full = listOf("apple", "orange", "banana")
	println(empty.ifEmpty { listOf("no", "values", "here") })
	// no values here
	println(full.ifEmpty { listOf("no", "values", "here") })
	// apple, orange, banana
}
```

## 6.1.9 컬렉션 나누기 chunked 와 windowed

연속적인 시간 값등을 처리하고 싶을 때가 있다.

슬라이딩 윈도우

```kotlin
val temperatures = listOf(27.7, 29.8, 22.0, 35.5 19.1)

// 3일간의 온도 평균을 구하고 싶다면 3인 슬라이딩 윈도우 사용 가능

fun main() {
	println(temperatures.windowed(3))
	// [27.7, 29.8, 22.0], [29.8, 22.0, 35.5], [22.0, 35.5 19.1]
	println(temperatures.windowed(3) {it.sum / it.size })
	// 26.5 , 29.09999, 25.53333
}
```

겹치지 않는 부분으로 나누고 싶을 때 chunked

```kotlin

fun main() {
	println(temperatures.chuncked(2))
	// [27.7, 29.8], [22.0, 35.5], [19.1]
	println(temperatures.chuncked(2) {it.sum})
	// 57.5 , 57.5, 19.1
}
```

## 6.1.10 컬렉션 합치기: zip

연관이 있는 두 컬렉션을 종합해야 할 때가 있을 것이다

```kotlin
val names = listOf("Joe", "Mary", "Jamie")
val ages = listOf(22, 31, 31, 44, 0)

fun main(){
	println(names.zip(ages))
	// [(Joe,22), (Mary,31), (Jamie,31)]
	
	println(ages.zip(names){age, name -> Person(name, age))	
	// [(Joe,22), (Mary,31), (Jamie,31)]
	
	println(ages zip names)	  //가능하지만 람다는 전달 못 함
}
```

결과 컬렉션은 두 컬렉션 중 짧은 컬렉션의 길이와 같다. zip은 두 입력 컬렉션 모두에 원소가 들어있는 인덱스에 해당하는 원소만 처리한다.

## 6.1.11 내포된 컬렉션의 원소 초리: flatMap과 flatten

취합한 결과가 List<List<>>일 때 이걸 펼치기 위해서 flatMap과 flatten을 사용할 수 있다.

```kotlin
class Book(val title: String, val authors: List<String>)

val library = listOf(
	Book("K I A", listOf("Isa", "Eli", "Aig")),
	Book("A K", listOf("Eck", "Isa")),
	Book("T TB P", listOf("Liu")),		
)

fun main() {
	val authors = library.map{it.authors}
	// List<List<String>> 이렇게 나온다
	
	// but
	val flatten = library.flatMap{it.authors}
	// "Isa", "Eli", "Aig", "Eck", "Isa", "Liu"
	
	// 반환할게 없고 단지 컬렉션의 컬렉션을 평평한 컬렉션으로 만들면 된다면
	listOfLists.flatten() 을 사용할 수 있다.
} 
```

## 6.2 지연 계산 컬렉션 연산: 시퀀스

map, filter 같은 함수들은 결과 컬렉션을 즉시 생성한다. 연쇄하면 매 단계마다 계산 중간 결과를 새로운 컬렉션 결과로 만든다. 자바 8 의 스트림과 비슷하게 시퀀스를 사용하면 중간 임시 컬렉션을 사용하지 않고 컬렉션 연산을 연쇄하는 방법을 제공한다

```kotlin
people.map(Person::name).filter { it.startsWith("A") }

// 대신
people.asSequence()
	.map(Person::name)
	.filter { it.startWith("A") }
	.toList()
```

각 단계 마다 컬렉션이 생성되지 않으며.

Sequence 안의 iterator라는 메서드를 통해 원소 값을 얻을 수 있다.

시퀀스의 원소는 lazy 계산된다. 따라서 중간 처리 결과를 만들지 않고 연산을 연쇄적으로 수행할 수 있다.

asSequence를 활용하여 시퀀스로 만들고 toList를 사용하여 리스트로 만든다.

원소를 순차적으로 접근할 때는 Sequence

인덱스로 접근할 때 List를 사용하면 좋다.

<aside>
💡

컬렉션 크기가 작으면 즉시 계산이 빠르지만 크기가 커지면 중간 원소 재밸열하는 비용이 커지기 때문에 지연계산이 낫다.

</aside>

## 6.2.1 시퀀스 연산 실행: 중간 연산과 최종 연산

시퀀스 연산은 **중간연산**과 **최종연산**으로 나뉜다

중간연산 : 다른 시퀀스를 반환, 항상 지연 계산이다. (최종연산이 호출되어야 계산됨)

최종연산 : 결과를 반환 (컬렉션, 원소, 수 등)

find에서 sequence가 아주 유용하다.

```kotlin
val age = listOf(1, 2, 3, 4, 5)
val sequence = age.asSequence()

age.map(it * it).find{it > 3} // 1, 4, 9, 16, 25를 만들고 find를 수행

// but
age.map(it * it).find{it > 3}  // 1, 4 에서 stop
```

## 6.2.2 시퀀스 만들기

generateSequence 함수를 사용해서 시퀀스를 만들 수도 있따.

```kotlin
fun main() {
	val naturalNumbers = generateSequence(0) { it + 1 }
	val numbersTo100 = naturalNumbers.takeWhilte { it <= 100 }
	println(numbersTo100.sum()) // 지연계산
	// 5050

}

fun File.isInsideHiddenDirectory() =
	generateSequence(this) { it.parentFile }.any { it.isHidden }

fun main() {
	val file = File("/Users/svtk/.HiddenDir/a.txt")
	println(file.isInsideHiddenDirectory())
	// 순차적으로 다음 원소를 계산한다.
}
```
