# Kotlin in Action 2/e 8장 정리  
기본 타입, 컬렉션, 배열

---

# 1. 기본 타입

## 1) 코틀린은 primitive 타입을 직접 노출하지 않는다
코틀린의 `Int`, `Double` 등은 객체처럼 보이지만, 실제로는 컴파일 시 상황에 따라 다르게 처리된다.

- 일반 변수 → primitive로 최적화  
- nullable (`Int?`) → wrapper(Integer)  
- 제네릭, 컬렉션 → boxing 발생  

즉, primitive를 숨기지만 **완전히 제거된 것은 아니다**.

---

## 2) nullable과 boxing

    val a: Int = 10
    val b: Int? = 10

- `Int` → primitive 가능  
- `Int?` → 객체 (null 표현 필요)

→ 성능과 메모리에 영향

---

## 3) 숫자 변환은 명시적

자바:
    int i = 10;
    long l = i;

코틀린:
    val i = 10
    val l = i.toLong()

→ 암묵적 변환 제거 = 타입 안정성 증가

---

## 4) 동등성 비교

    a == b   // 값 비교
    a === b  // 참조 비교

→ nullable / boxing 상황에서 `===` 주의

---

## 5) Any

- 자바 Object와 동일
- primitive 넣으면 boxing 발생

---

## 6) String / Char / Boolean

- String → immutable (자바와 동일)
- Char → 숫자와 자동 변환 안됨
- Boolean → nullable 가능하지만 조건문에서 null 불가

---

# 2. 컬렉션

## 1) 읽기 전용 vs 가변

    val list: List<Int>
    val mutable: MutableList<Int>

- List → read-only
- MutableList → 수정 가능

---

## 2) 진짜 immutable은 아님

    val list: List<Int> = mutableListOf(1, 2, 3)

- 실제 객체는 mutable
- List는 인터페이스일 뿐

---

## 3) 자바와 차이

자바:
- 읽기/쓰기 구분 없음

코틀린:
- 타입으로 분리

!!JVM에서는 동일 → 자바에서 변경 가능!!

---

## 4) nullability

    List<Int?>   // 요소 nullable
    List<Int>?   // 리스트 nullable

→ 완전히 다른 의미

---

## 5) 컬렉션과 boxing

    List<Int>

- 내부적으로 Integer 사용
- 모든 요소 boxing

→ 대량 데이터에서 성능 영향

---

# 3. 배열

## 1) 배열 생성

    val arr = arrayOf(1, 2, 3)
    val arr2 = Array(5) { it * 2 }

---

## 2) primitive 배열

    val arr = IntArray(5)

- int[]로 컴파일
- boxing 없음
- 성능/메모리 효율 좋음

---

## 3) 배열 vs 컬렉션

배열:
- 저수준
- 성능 중심

컬렉션:
- 고수준
- 사용성 중심

→ 일반적으로 컬렉션 사용, 성능 필요 시 배열

---

## 4) vararg + spread

    fun sum(vararg nums: Int)

    val arr = intArrayOf(1, 2, 3)
    sum(*arr)

- `*` → spread operator
- 배열 복사 비용 발생 가능

---

## 5) 자바 interop

    fun main(args: Array<String>)

→ 자바 String[]와 동일

---

# 4. 자바 차이

1. primitive를 숨기지만 존재함  
2. read-only 컬렉션은 JVM에서 강제되지 않음  
3. boxing 여전히 존재  
4. 암묵적 타입 변환 제거
