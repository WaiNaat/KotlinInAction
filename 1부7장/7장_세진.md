# 7장. 널이 될 수 있는 값

## 1. null 가능/불가능 타입을 컴파일러가 처리하는 방식

- **null 가능성 자체를 타입에 포함시킨다**

```kotlin
var a: String = "abc"   // null 불가능
var b: String? = null   // null 가능
```

### 컴파일러 동작 방식

* `String` → 절대 null 올 수 없음 (컴파일 단계에서 검사)
* `String?` → null 가능 → 접근 시 반드시 안전 처리 요구

```kotlin
val length = a.length   // OK
val length2 = b.length  // 컴파일 에러
```

- 즉, NPE를 "런타임"이 아니라 "컴파일 타임"에 막음

---

## 2. 안전한 호출 연산자 `?.`

null이면 그냥 null 반환, 아니면 호출

```kotlin
val len = b?.length
```

### 특징

* 체이닝 가능

```kotlin
person?.address?.city
```

* 결과 타입도 nullable

```kotlin
Int?
```

---

## 3. 엘비스 연산자 `?:`

null일 때 기본값 제공

```kotlin
val len = b?.length ?: 0
```

### 중요한 포인트

* return / throw도 가능

```kotlin
val name = person.name ?: return
val value = input ?: throw IllegalArgumentException()
```

---

## 4. 안전한 타입 캐스트 `as?`

캐스트 실패 시 null 반환

```kotlin
val obj: Any = "hello"
val str: String? = obj as? String
```

### 일반 캐스트와 차이

```kotlin
obj as String   // 실패 시 ClassCastException
obj as? String  // 실패 시 null
```

---

## 5. 널 아님 단언 `!!`

"절대 null 아님"을 강제로 보장

```kotlin
val len = b!!.length
```

### 위험성

* null이면 즉시 NPE 발생

---

## 6. `let` 함수

```kotlin
b?.let {
    println(it.length)
}
```

### 특징

* null 아닐 때만 실행
* `it`으로 접근

### 체이닝

```kotlin
person?.email?.let { sendEmail(it) }
```

👉 **if (x != null) 대신 사용하는 함수형 스타일**

---

## 7. 스코프 함수 총정리

| 함수    | this / it | 반환값    | 용도           |
| ----- | --------- | ------ | ------------ |
| let   | it        | 마지막 결과 | null 처리, 체이닝 |
| run   | this      | 마지막 결과 | 객체 초기화 + 결과  |
| with  | this      | 마지막 결과 | 여러 호출 묶기     |
| apply | this      | 자기 자신  | 객체 설정        |
| also  | it        | 자기 자신  | 부가 작업 (로그 등) |

---

👉 객체 설정

```kotlin
val user = User().apply {
    name = "kim"
    age = 20
}
```

👉 null 처리

```kotlin
user?.let { ... }
```

👉 계산 결과 반환

```kotlin
val result = user.run { name.length }
```

👉 사이드 이펙트

```kotlin
user.also { println(it.name) }
```

👉 단순 블록 묶기

```kotlin
with(user) {
    println(name)
}
```

## 8. 지연 초기화 프로퍼티 `lateinit`

```kotlin
lateinit var service: Service
```

### 특징

* 나중에 초기화 가능
* non-null 타입 유지
* 주로 Android (ViewBinding, DI)에서 사용

```kotlin
service = Service()
```

### 제약

* var만 가능
* primitive 불가
* 초기화 전에 접근하면 예외

---

## 9. 제네릭에서의 null (`T는 Any?`)

기본적으로:

```kotlin
fun <T> foo(value: T)
```

👉 여기서 T는 **nullable 포함**

즉:

```kotlin
foo(null)  // 가능
```

### null 금지하려면

```kotlin
fun <T : Any> foo(value: T)
```

👉 T를 non-null로 제한

---

## 10. 플랫폼 타입

자바에서 온 타입:

```java
String name;
```

코틀린에서는:

```kotlin
val name: String
val name: String? //둘 다 가능하도록.
```

### 특징

* nullable인지 non-null인지 모름
* 컴파일러가 체크 안함

```kotlin
val len = name.length  // 가능하지만 위험
```

👉 **NPE 발생 가능**

---

### 그럼 어떻게 쓰지?

```kotlin
val safeName: String? = javaName
val safeName2: String = javaName ?: "default"
```

👉 항상 명시적으로 변환해서 사용
