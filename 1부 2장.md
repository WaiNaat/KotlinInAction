3. 아래 `eval` 함수에서 스마트 캐스트가 동작하는 이유는? (2.3.6 p.109) (김세진)
    
    ```kotlin
    fun eval(e: Exp): Int {
        if (e is Num) return e.num  // 캐스팅 없이 e.num 접근
        if (e is Sum) return eval(e.right) + eval(e.left)
        throw IllegalArgumentException("지원하지 않는 타입")
    }
    
    ```
    
    - e is Num, e is Sum 에서 타입 검사가 일어났기 때문에, 컴파일러가 뒤에 올 e의 타입을 100% 보장할 수 있기 때문에, 내부 값에도 접근이 가능해진다.

3. 코틀린에서 임의의 클래스가 범위(`..` )가 될 수 있는 조건은 무엇일까? (김세진)
(p.118, p.121)
    - 비교가 가능한 클래스라면 클래스의 인스턴스 객체를 사용해 범위를 만들 수 있다.
    - 그 비교가 가능한 클래스는, Java.lang.Comparable 인터페이스를 구현한 클래스이다
    
    3-1. Q) 반복문 내에서 이터레이션을 할 수 있는 조건은 무엇일까?
    
    - (한줄요약) Kotlin의 `for`문은 `iterator()`를 호출하여 `Iterator`를 얻고 `hasNext()`와 `next()`를 반복 호출하는 문법적 설탕이다.
        
        ```kotlin
        for (x in list) {
            println(x)
        }
        
        // 위 코드는
        // 컴파일러에 의해
        // 아래와 같이 바뀐다.
        
        val iterator = list.iterator()
        
        while (iterator.hasNext()) {
            val x = iterator.next()
            println(x)
        }
        ```
        
    - for는 iterator 기반으로 동작한다.
        - `iterator`
            - 다음 원소를 하나씩 꺼내주는 객체
            - 즉, 컬렉션을 순차적으로 탐색하는 방법을 정의한 인터페이스
            - Kotlin의 Iterator 인터페이스
                
                ```kotlin
                interface Iterator<out T> {
                    operator fun next(): T
                    operator fun hasNext(): Boolean
                }
                ```
                
        - List, Set, Map, Int 같은 경우는 iterator 를 제공한다.
        - LocalDate는 iterator를 제공하지 않지만, iterator를 직접 구현하면 반복문에서 사용 가능하다.
            
            ```kotlin
            operator fun ClosedRange<LocalDate>.iterator(): Iterator<LocalDate> =
                object : Iterator<LocalDate> {
            
                    var current = start
            
                    override fun hasNext() = current <= endInclusive
            
                    override fun next(): LocalDate {
                        val result = current
                        current = current.plusDays(1)
                        return result
                    }
                }
            ```
